# Purpose

The Homepage is the physical manifestation of the Marzoq product.

It is not a landing page.

It is not a marketing homepage.

It is not a corporate website.

It is the product's primary interface.

Every visual element exists to communicate one central idea:

Marzoq helps businesses become discoverable inside AI.

The Homepage explains this through interaction before explanation.

---

# Canonical Principle

The Homepage consists of exactly one viewport.

Height

100vh

Width

100vw

No vertical scrolling.

No horizontal scrolling.

No secondary hero.

No stacked sections.

No hidden continuation below the fold.

Everything the visitor needs to understand the product exists inside the first screen.

Scrolling is reserved for content pages.

Never for the Homepage.

---

# Mission

The Homepage has one objective.

Transform confusion into understanding within seconds.

The visitor should understand:

What Marzoq is.

Who it is for.

Why it matters.

Why it is different.

Without navigating away.

# Viewport Philosophy

The viewport is treated as a digital stage.

Not a document.

Every component shares the same visual space.

No section begins after another.

Everything coexists.

Hierarchy is achieved through attention.

Not scrolling.

---

# Structural Hierarchy

The Homepage is composed of six permanent systems.

These systems coexist inside the same viewport.

They are layered, not stacked.

They are rendered simultaneously rather than sequentially.

Their order defines visual hierarchy, not document flow.

| Layer | System | Role |
| ----- | ------ | ---- |
| 1 (back) | Background | Environmental presence |
| 2 | Ecosystem | Context |
| 3 | Compass | Direction — the visual origin |
| 4 | Content Layer | Explanation |
| 5 | Header | Orientation |
| 6 (front) | Footer | Completion |

The Header is not above the Compass. The Footer is not below the Content Layer. Nothing in this structure implies vertical sections. The Homepage is one composition, not multiple regions arranged from top to bottom.

Every system is always present.

No system enters later.

No system is conditionally rendered.

---

# Background

Purpose

Provide environmental presence.

Contains

* background color
* environmental grid
* orbital rings
* subtle depth

The Background is passive.

It never competes for attention.

---

# Header

Purpose

Orientation.

Contains

* logo
* primary navigation
* primary CTA

The Header remains visible.

It never becomes sticky.

It never collapses.

It is part of the viewport composition.

---

# Ecosystem

Purpose

Context.

The Ecosystem surrounds the Compass.

It visualizes:

AI Platforms

↓

Human Questions

↓

Digital Relationships

The Ecosystem never replaces content.

It reinforces meaning.

---

# Compass

Purpose

Direction.

The Compass is the visual origin of the Homepage.

Everything else is positioned relative to it.

Its specification is defined separately.

This document only defines its role within the page.

---

# Content Layer

Purpose

Explanation.

The Content Layer introduces Marzoq using minimal language.

It should answer:

What

↓

Why

↓

Who

↓

Action

The Content Layer should never dominate the visual experience.

Reading follows observation.

Not the opposite.

---

# Footer

The Footer exists inside the viewport.

It is not a separate page section.

Purpose

Completion.

It contains:

* legal links
* secondary navigation
* copyright
* small utilities

Its visual weight is intentionally low.

The Footer quietly completes the composition.

---

# Single Composition Principle

The Homepage should be perceived as one composition.

Not six independent components.

Every element should appear intentionally positioned.

The visitor should never mentally divide the page into sections.

The Homepage behaves as one coherent instrument.

---

# Information Priority

Visual understanding always precedes textual understanding.

Priority order

Compass

↓

Interaction

↓

Ecosystem

↓

Headline

↓

Supporting Text

↓

CTA

↓

Navigation

↓

Footer

Any implementation that changes this order is considered non-canonical.
