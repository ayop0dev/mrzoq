# Purpose

The Ecosystem represents the digital environment surrounding the Compass.

It is not decoration.

It is not a background.

It is not an animation layer.

It visualizes the world in which businesses, AI systems, search intent, and digital authority interact.

The Ecosystem exists to make an invisible concept visible.

The Compass gives direction.

The Ecosystem provides context.

Together they explain Marzoq before a single paragraph is read.

---

# Principles

The Ecosystem must always feel:

* alive
* intelligent
* lightweight
* spacious
* calm
* intentional
* continuously evolving

It must never feel:

* chaotic
* crowded
* random
* decorative
* playful
* futuristic
* game-like

Movement communicates relationships.

Never excitement.

---

# Responsibilities

The Ecosystem owns:

* environmental context
* AI platform representation
* search intent representation
* spatial depth
* visual exploration
* focus feedback

The Ecosystem never owns:

* navigation
* page layout
* business content
* typography hierarchy
* user interface controls

The Compass remains the primary actor.

The Ecosystem always supports it.

Each token represents one recognizable platform.

Examples

* ChatGPT
* Claude
* Gemini
* Perplexity
* Copilot
* Midjourney

The category does not.

Platform Tokens represent intelligence.

Not companies.

Each Platform Token owns:

* platform icon
* platform name
* orbital position
* focus state

Platform Tokens never own:

* search queries
* navigation
* interaction logic
* business messaging

# Visual Structure

Each Platform Token consists of:

Container

↓

Platform Icon

↓

The label always appears second.

# Query Pills

Query Pills represent real user intent.

They visualize questions users ask AI systems.

They communicate discoverability.

Not keywords.

Every Query Pill should read like a genuine human question.

Never like SEO text.

---

# Query Guidelines

Every query should feel:

natural

specific

commercially realistic

geographically believable

Examples

"Best ERP company in Saudi Arabia"

"أفضل شركة تشطيب في الرياض"

"أفضل مستشفى أطفال في جدة"

Queries should always represent real search behaviour.

Marketing slogans are forbidden.

---

# Query Ownership

Each Query Pill owns:

query text

orbital position

focus state

visual priority

Nothing else.

---

# Object Independence

Every floating object behaves independently.

Objects do not know about one another.

Each object calculates only:

its own position

its own motion

its own focus

Global coordination belongs to the Ecosystem.

Never individual objects.

---

# Orbital Architecture

The Ecosystem is not a particle system.

It is a structured orbital system.

Objects exist inside invisible orbital regions.

The regions create order without visible constraints.

The user should never perceive rigid geometry.

Only natural balance.

---

# Orbital Layers

The Ecosystem contains three conceptual layers.

Far Layer

↓

Middle Layer

↓

Near Layer

Each layer owns different visual weight.

Each layer moves independently.

Layer responsibilities never overlap.

---

## Far Layer

Purpose

Environmental depth.

Contains the least important objects.

Characteristics

* lowest opacity
* slowest movement
* largest orbital radius
* weakest parallax

The Far Layer creates atmosphere.

Nothing more.

---

## Middle Layer

Purpose

Primary exploration.

Most Platform Tokens belong here.

This is the busiest layer.

It communicates the active AI landscape.

Objects remain readable.

Never dominant.

---

## Near Layer

Purpose

Human intent.

Most Query Pills belong here.

Objects occasionally become the primary visual focus when aligned by the Compass.

The Near Layer feels closest to the visitor.

Yet never obstructs the Compass.

---

# Spatial Rules

Objects always maintain visual breathing room.

Minimum spacing is mandatory.

Objects may approach.

They never collide.

Visual overlap is forbidden.

The Ecosystem should always feel open.

Density is intentionally low.

# Component States

The Ecosystem has four canonical states.

Every floating object exists in one of these states.

No additional states should be introduced.

---

## State 01

Ambient

The default state.

Every object quietly drifts within its orbital region.

Characteristics

* low visual priority
* soft opacity
* reduced contrast
* continuous movement
* no emphasis

The Ecosystem should feel alive before the user interacts.

Never static.

---

## State 02

Attention

Occurs when the user's pointer moves through the viewport.

The Ecosystem acknowledges attention indirectly.

Objects do not chase the pointer.

Instead,

environmental depth subtly shifts.

Characteristics

* gentle parallax
* maintained orbital motion
* unchanged hierarchy

The user influences the environment.

Never controls it.

---

## State 03

Focus

Occurs when the Compass Needle aligns with an object.

The object becomes temporarily important.

Characteristics

* increased opacity
* reduced blur
* restored color
* slight scale increase
* elevated visual layer

Only one object should normally occupy the strongest focus.

The transition into Focus must remain smooth.

No sudden pop.

No flashing.

---

## State 04

Reduced Motion

Activated through

prefers-reduced-motion.

Characteristics

* orbital drift disabled
* parallax disabled
* breathing disabled
* focus preserved

Objects remain readable.

Relationships remain understandable.

Identity survives without motion.

---

# Orbital Behaviour

Objects continuously drift.

Drift is not random.

Every object follows its own invisible orbital path.

Each path is deterministic.

Given the same starting state,

the object always follows the same motion.

The Ecosystem should feel designed.

Never simulated.

---

# Orbital Independence

Every floating object owns:

its own

* orbital radius
* orbital phase
* orbital speed
* orbital offset

No object inherits movement from another.

The Ecosystem coordinates.

Objects do not communicate.

---

# Motion Hierarchy

Priority

1.

Compass Alignment

↓

2.

Focus Recognition

↓

3.

Orbital Drift

↓

4.

Environmental Parallax

↓

5.

Background Presence

Higher-priority motion always wins.

If performance degrades,

lower-priority motion is reduced first.

---

# Orbital Speed

Motion must remain subtle.

Objects should never appear to orbit visibly.

Instead,

users should gradually discover that nothing is actually static.

Movement should be perceived after observation.

Not immediately.

---

# Drift Behaviour

Orbital drift communicates life.

Not navigation.

Objects never travel large distances.

They remain close to their canonical positions.

The Homepage composition must remain visually stable.

---

# Environmental Parallax

Every object owns an independent depth value.

Depth affects:

horizontal displacement

vertical displacement

perceived distance

Objects closer to the user move slightly more.

Far objects move less.

Parallax exists to reinforce spatial depth.

Not spectacle.

---

# Focus Behaviour

Focus is determined by Compass alignment.

Never by pointer hovering.

This is one of the defining interaction principles of Marzoq.

The user understands that:

Direction

creates

Recognition.

Not proximity.

---

# Visual Transition

Entering Focus

Objects gradually:

increase opacity

restore saturation

remove blur

restore scale

raise stacking order

Leaving Focus

The exact reverse occurs.

Transitions should never appear binary.

Everything fades naturally.

---

# Platform Token Behaviour

Platform Tokens remain calm.

They never pulse.

They never rotate.

They never animate independently.

Their motion comes only from:

orbital drift

environmental parallax

focus transition

Platform Tokens represent stable AI systems.

They should feel dependable.

---

# Query Pill Behaviour

Query Pills represent human curiosity.

They are slightly more dynamic than Platform Tokens.

However,

their movement remains restrained.

Query Pills never bounce.

Never wobble.

Never jitter.

They quietly exist within the ecosystem.

---

# Alignment Logic

The Compass Needle defines the active direction.

Objects compare their own angular position against the Needle.

When alignment falls inside the accepted threshold,

Focus is granted.

Focus is temporary.

The Ecosystem constantly reevaluates alignment.

The Compass never requests Focus.

Objects determine it independently.

# Responsive Behaviour

The Ecosystem adapts to every supported viewport.

Its density changes.

Its identity does not.

---

## Desktop

Desktop is the canonical experience.

All layers remain active.

Platform Tokens and Query Pills surround the Compass.

Maximum environmental depth is preserved.

---

## Tablet

Reduce environmental density.

Increase spacing.

Reduce motion intensity.

Compass remains visually dominant.

Objects remain readable.

---

## Mobile

Mobile prioritizes clarity.

Reduce the number of simultaneously visible objects.

Preserve all categories.

Not all instances.

The user should still perceive:

AI Platforms

↓

Human Questions

↓

Compass Guidance

without visual overload.

---

# Density Rules

The Ecosystem should never attempt to fill empty space.

Negative space is intentional.

Whitespace communicates confidence.

The experience should always feel breathable.

Not busy.

# Component API

The Ecosystem exposes a minimal public interface.

Inputs

* Compass direction
* viewport dimensions
* reduced motion preference
* active focus target

Internal State

* orbital positions
* orbital phases
* depth values
* focus state
* drift offsets

Outputs

* focused object
* alignment feedback
* visual emphasis

The Ecosystem contains no business logic.

No CMS content.

No routing.

---

# Performance Requirements

The Ecosystem occupies the first screen.

Its performance is critical.

Rendering must remain lightweight.

Every object should animate using transforms only.

Avoid layout recalculation.

Avoid repaint-heavy effects.

Opacity and transforms are preferred.

Blur should remain static whenever possible.

Movement must be GPU-friendly.

---

# Loading Behaviour

The Ecosystem exists immediately.

Objects are already present.

No staggered entrance animation.

No delayed reveal.

The visitor enters a living environment.

Not an animated presentation.

---

# Failure Behaviour

If JavaScript is unavailable:

Objects remain visible.

Platform Tokens remain readable.

Query Pills remain readable.

Orbital positioning remains intact.

The Compass still communicates direction.

The Homepage continues to explain the product.

Motion enriches understanding.

It never creates understanding.
